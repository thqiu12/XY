## 1. 最重要要件 — 銀行審査対応

銀行の審査担当者は、サイト記載情報を **登記事項証明書 / 定款** と突き合わせる。以下を厳守する。

- 会社概要は登記・定款と**逐語一致**(商号・所在地・代表者・設立日・資本金・法人番号)。
- 事業内容は**登記目的の範囲内**で記述(超範囲経営に見えてはならない)。
- **虚偽・誇大・架空の顧客ロゴ/数値を作らない**(CLAUDE.md 準拠)。
- 連絡先(電話・メール)は**実在し到達可能**であること。
- 「事業の実態」が伝わること(代表の経歴・サービス・実績)。

## 5. サイト構成

技術: 静的サイト(HTML/CSS、ビルド不要)。日本語主体。レスポンシブ(モバイル重視)。

- `index.html` — ヒーロー / 課題提起 / サービス(6) / XYの強み / 代表メッセージ / 導入事例 / CTA
- `company.html` — 会社概要(審査の中核。登記準拠の全項目テーブル + 事業内容 + 地図リンク)
- `contact.html` — お問い合わせ(電話・メール・所在地 + 簡易フォーム mailto ベース)
- `privacy.html` — プライバシーポリシー(信頼性向上、任意だが推奨)
- `css/style.css` — 全ページ共通スタイル
- `assets/` — SVG ロゴ「XY」、favicon、OGP 画像
- `CNAME`(xy-cloud.org)、`robots.txt`、`sitemap.xml`

## 6. デザイン方針

- 配色: 靖蓝(信頼)+ 深墨 + 暖灰。教育・コンサル業界の堅実な専門性。過度な装飾を避ける。
- ロゴ: 純 SVG の「XY」字標。
- 書体: Noto Sans JP / Hiragino 系のシステムフォント。商務感。
- アクセシビリティ: コントラスト確保、セマンティック HTML、alt 属性。

## 7. 導入事例の扱い

現時点では**匿名・中性の枠組みで占位**(例:「都内日本語学校A様」)。具体的な成果数値は捏造しない。実在事例の提供後に差し替える。
